Did you ever imagine that how these graphics and games are run on a web-browser? What is the main technology behind it?

In previous days we used WebGL for this task. WebGL(Web graphics Library) is JavaScript API which is used to render the 3-dimensional objects, 2-dimensional objects and Graphics on the Web-Browser

Why 3JS when we have WebGL

WebGL is very low level system that only draws basic objects like point, square and line. To do any meaningful thing with WebGL generally requires quite a bit of code and that is where Three.js comes in.

What is Three.js?

Three.js is a cross-browser JavaScript library and application programming interface (API) used to create and display animated 3D computer graphics in a web browser. Three.js uses WebGL.

Three.js was released by Ricardo Cabello in April 2010.

Why Three.js?

Three.js makes WebGL - 3D in the browser - very easy. While a simple cube in raw WebGL would take hundreds of lines of code, a Three.js equivalent is just a few lines.

Since we are using JavaScript, we can also interact with other HTML elements easily

Advantages of Three.js

Since Three.js is open source, we can easily watch the source code and understand the functionality of the code (functions).

When we use WebGL for graphics, it doesn't support most browsers, but Three.js supports most browsers.

It doesn't require any third-party plugin to run the code.

You just need to work with JavaScript and HTML.

How To Include Three.js in Your Projects?

Download the Three.js library from the official website.

Use a CDN to include the Three.js library in your HTML file.

Use Node.js to install Three.js as a dependency in your project.

How a 3D Scene Works?

In a 3D scene, there are several key components:

Scene: Contains all the objects, lights, and cameras in the scene.

Camera: Determines the perspective and view of the scene.

Object: Represents the 3D objects in the scene, combining
and material to define their appearance and properties.

Lighting: Light up the scene.

Audio: Plays the sounds/music.

Renderer: Renders the scene onto the canvas.

How to Create a 3D Scene?

To create a 3D scene in Three.js, you need to follow these steps:

Create a scene, camera, and renderer.

Add objects, lights, and cameras to the scene.

Set up the camera position and perspective.

Render the scene using the renderer.

Update the scene in the animation loop.

How to Create a 3D Object?

To create a 3D object in Three.js, you need to follow these steps:

Create a geometry to define the shape of the object.

Create a material to define the appearance of the object.

Combine the geometry and material to create a mesh.

Add the mesh to the scene.

How to Load Models?

To load 3D models into a Three.js scene, you can use GLTFLoader, FBXLoader, or OBJLoader. The models loaded can be animated and interacted with like any other object in the scene.

How to Animate a 3D Object?

To animate a 3D object in Three.js, you need to follow these steps:

Update the object's position, rotation, or scale in the animation loop.

Use the requestAnimationFrame() method to update the animation.

You can use the TWEEN.js library to create complex animations.

How to Add Interaction?

There are several interactions type in Three.js, each with its own unique usage.

The Raycaster class detects mouse clicks on objects.

the OrbitControls class to allow users to rotate the camera field of view around the scene, and even move the camera.

The PointerLockControls class to allow users to move the camera around the scene using the mouse or keyboard controls.

How to Add Physics?

To add physics to a 3D scene in Three.js, use the Cannon.js library to create physics simulations.

Create a world, bodies, and constraints to define the physics scene.

Update the physics simulation in the animation loop.

How to Add Audio?

To add audio to a 3D scene in Three.js, you can use the Web Audio API to load and play audio files.

Create an AudioListener and Audio object to control the audio.

Add the Audio object to the scene and play the audio.

How to Add Post-Processing?

To add post-processing effects to a 3D scene in Three.js, there are serveral styles:

The EffectComposer class creates post-processing effects.

The RenderPass and ShaderPass classes define the effects.

Afterwards, add the effects to the composer and render the scene.
